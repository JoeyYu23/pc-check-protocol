# 买家核验清单

卖家运行验机工具后，你会收到一个 ZIP 文件。以下是完整的核验指南，适用于二手台式机和笔记本，涵盖 GPU、CPU、内存、硬盘各项检测结果的解读。

---

## 解压后文件说明

| 文件 | 看什么 |
|------|--------|
| `system_info.txt` | 显卡型号、VRAM、驱动版本、主板、内存、CPU |
| `summary.txt` | 快速摘要 |
| `test_summary.txt` | 各测试通过/跳过/失败状态 |
| `dxdiag_output.txt` | DirectX 诊断，含详细显卡和系统信息 |
| `furmark_log.txt` | FurMark GPU 烤机结果 |
| `occt_log.txt` | OCCT 显存测试记录 |
| `cpu_stress_log.txt` | CPU 压力测试记录 |
| `memory_test_log.txt` | 内存稳定性测试记录 |
| `disk_health.txt` | 硬盘 SMART 健康报告 |
| `thermal_stress_log.txt` | 散热综合评估记录 |
| `hwinfo_sensors.csv` | 传感器时序数据（用 Excel 打开分析温度曲线）|
| `screenshot_*.png` | 测试前中后截图 |

---

## 核验步骤

### 1. 确认 CPU 型号

打开 `system_info.txt`，找到 `[CPU]` 部分：

```
名称:       Intel Core i9-13900K
核心数:     24 核 (8P + 16E)
基础频率:   3.00 GHz
```

**核验要点：**
- 型号是否与卖家描述一致（如 i9-13900K vs i9-13900KF，5800X vs 5800X3D）
- 核心数是否与该型号标称一致

**常见作弊手法：**
- 低端型号的 CPU，用软件工具修改显示名称（核心数无法造假）
- 要求卖家在视频中打开 CPU-Z 验证型号和核心数

---

### 2. 确认 GPU 型号

打开 `system_info.txt`，找到 `[显卡 GPU]` 部分：

```
名称:       NVIDIA GeForce RTX 4090
显存 (WMI): 24576 MB (24.0 GB)
驱动版本:   31.0.15.6079
PNP设备ID:  PCI\VEN_10DE&DEV_2684...
```

**核验要点：**
- 名称是否与卖家描述一致（如 RTX 4090 vs RTX 4090D，RTX 4080 vs RTX 4080 Super）
- VRAM 大小是否与该型号标称一致（见下方规格参考表）
- `VEN_10DE` = NVIDIA（正确）；`DEV_XXXX` 编号决定具体型号
- 驱动版本格式是否正常

---

### 3. FurMark GPU 稳定性

打开 `furmark_log.txt`，检查：

```
测试结果: 通过 (全程稳定运行)
计划时长: 300 秒
实际时长: 301 秒
退出代码: 0
```

**正常：** 测试结果为"通过"，实际时长接近计划时长，退出代码为 0

**异常：**
- 测试结果为"异常 (提前退出)"
- 实际时长远小于计划时长（卡死或崩溃）
- 退出代码非 0（应用崩溃）

---

### 4. CPU 压力测试

打开 `cpu_stress_log.txt`：

**正常：** 最终结果为 `PASS`，测试完整跑完

**异常：** 结果为 `WARN` 或提前终止（CPU 过热、蓝屏、崩溃）

---

### 5. 内存稳定性

打开 `memory_test_log.txt`：

**正常：** 最终结果为 `PASS`，错误计数为 0

**异常：**
- 结果为 `WARN`
- `错误计数` > 0（内存颗粒可能有问题）
- 测试中途中断

**红旗：** 内存测试出错是严重问题，可能是内存条故障或兼容性问题。

---

### 6. 硬盘 SMART 健康度（重要）

打开 `disk_health.txt`，重点看每块硬盘的以下字段：

#### 硬盘健康参考标准

| 指标 | 健康 | 注意 | 异常/红旗 |
|------|------|------|---------|
| 健康状态 | Healthy | Warning | Unhealthy / Failed |
| 通电时间 | < 20000 小时 | 20000-30000 小时 | > 30000 小时 |
| SSD 磨损程度 (Wear) | < 50% | 50-80% | > 80% |
| 温度 | < 50°C | 50-55°C | > 55°C |
| 未纠正读取错误 | 0 | — | > 0 |
| 未纠正写入错误 | 0 | — | > 0 |

**通电时间换算参考（500GB-2TB 主流硬盘）：**
- 1 年 ≈ 8760 小时（24小时开机）
- 20000 小时 ≈ 2.3 年连续使用，或 5-8 年正常使用
- SSD 主流 TBW 寿命：500GB SSD 约 150-300 TBW，高端 1TB 约 600 TBW+

**SSD 磨损程度解读：**
- 磨损程度 (Wear) 由 Windows 从 SMART 属性换算，0% = 全新，100% = 达到寿命上限
- 购买二手 SSD 时，建议 Wear < 50%，价格应相应反映磨损程度

---

### 7. 温度分析（HWiNFO CSV）

用 Excel 或 WPS 打开 `hwinfo_sensors.csv`，查找以下列：

| 传感器列名 | 含义 |
|-----------|------|
| `GPU Temperature` | GPU 核心温度 |
| `GPU Hot Spot Temperature` | GPU 结温 (Hot Spot) |
| `CPU Package` 或 `CPU Temperature` | CPU 整体温度 |
| `GPU Fan Speed` | GPU 风扇转速 |
| `CPU Fan Speed` | CPU 风扇转速 |
| `GPU Power` | GPU 功耗 |

#### 各硬件温度正常范围（压力测试中）

| 硬件 | 正常 | 注意 | 异常 |
|------|------|------|------|
| GPU 核心温度 | 70-85°C | 85-95°C | 95°C+ |
| GPU 结温 (Hot Spot) | 80-100°C | 100-108°C | 108°C+ |
| CPU 温度（高端风冷/水冷） | 70-85°C | 85-95°C | 95°C+ |
| CPU 温度（笔记本） | 80-95°C | 95-100°C | 100°C+ |
| GPU 显存温度 (GDDR6X) | 80-100°C | 100-108°C | 108°C+ |
| GPU 显存温度 (GDDR6) | 60-80°C | 80-90°C | 90°C+ |

**注意：** 笔记本 CPU 温度偏高属正常现象（散热空间限制），关键是看是否触发降频（Clock Frequency 列是否持续下降）。

---

### 8. OCCT 显存测试

打开 `occt_log.txt` 和对应截图：

**正常：** 测试完成，截图中显示 **0 errors**

**异常：**
- Errors 数量 > 0（显存存在故障颗粒）
- 测试中途崩溃或提前退出

---

### 9. 截图核验

检查 `screenshot_*.png` 文件时间戳是否连续、符合测试流程。

## 红旗汇总

以下任意一条出现，建议重新协商价格或拒绝交易：

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| CPU/GPU 名称与描述不符 | 严重 | 可能是低端型号改名 |
| VRAM 大小与标称不符 | 严重 | 非正品或显存损坏 |
| FurMark 提前退出 | 严重 | 显卡存在稳定性问题 |
| OCCT 出现 Errors | 严重 | 显存颗粒故障 |
| CPU 压力测试失败 | 严重 | CPU 或主板可能有问题 |
| 内存测试出错 | 严重 | 内存条故障 |
| 硬盘健康状态非 Healthy | 严重 | 立即备份数据，硬盘濒临故障 |
| 未纠正读/写错误 > 0 | 严重 | 硬盘坏道，有数据损失风险 |
| SSD 磨损 > 80% | 中等 | 寿命有限，应大幅压价 |
| 通电时间 > 30000 小时 | 中等 | 高度老化，故障率上升 |
| GPU/CPU 温度持续 95°C+ | 中等 | 散热系统可能需要维护 |
| 风扇转速为 0（烤机中） | 严重 | 风扇故障或被禁用 |
| 截图时间和文件时间不匹配 | 可疑 | 可能造假，要求现场视频验机 |
| 功耗远低于正常值 | 中等 | 可能有功耗限制导致性能下降 |

---

## 防伪建议

验机工具可以提高门槛，但无法完全防止造假。建议：

1. **视频验机**：要求卖家开视频，全程观看操作，确认截图是实时生成
2. **现场验机**：条件允许时去卖家处当面验机
3. **查序列号**：GPU-Z 里的序列号（如果显示）可以通过 NVIDIA 官网核实
4. **交叉验证**：`dxdiag_output.txt` 里的信息应与 `system_info.txt` 一致
5. **保留记录**：妥善保存验机 ZIP，作为日后维权的凭证

## 关于型号辨别的典型案例：RTX 4090 vs RTX 4090D

RTX 4090D 是专为中国大陆市场发行的出口管制版本（削减 CUDA 核心），是一个需要特别注意的型号混淆案例：

| 特征 | RTX 4090 | RTX 4090D |
|------|----------|-----------|
| CUDA 核心 | 16384 | 14592 |
| GPU 核心 | AD102-300 | AD102-301 |
| PNP DEV ID | 2684 | 2685 |
| 性能差异 | 基准 | 约低 11% |
| 功耗 | 450W TDP | 425W TDP |

在 `system_info.txt` 的 `PNP设备ID` 中，`DEV_2684` = 4090，`DEV_2685` = 4090D。如果卖家将 4090D 标价为 4090，这是价格欺诈。

其他同类型案例：RTX 3080 10GB 与 3080 12GB、RTX 4070 与 4070 Super 等，核验方法相同——对照上方规格表核实 VRAM 大小和 DEV ID。
